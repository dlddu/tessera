# 테스트 문서: 상태 복원

- **Test ID**: T-4
- **대상 PRD**: PRD-4 (`tessera-prd-state-restoration.md`)

## 검증 대상 AC

- AC4.1: 백엔드 종료 시 편집기 상태 복원
- AC4.2: 백엔드 종료 시 Claude Code 세션 상태 복원
- AC4.3: 백엔드 종료 시 터미널 마지막 화면 복원
- AC4.4: 앱 종료 후 브라우저 탭별 URL 복원
- AC4.5: 호스트 측 영속 저장 (backend/app 수명과 독립)
- AC4.6: 복원 상태와 재연결 간 일관성 처리

## 테스트 시나리오

### 시나리오 1: 편집기 복원 (백엔드 강제 종료)

- **사전 조건**: 편집기에 미저장 변경(여러 파일, 커서 위치)이 있는 상태
- **실행 단계**: backend(컨테이너/호스트 프로세스)를 강제 종료
- **기대 결과**: 열린 파일·미저장 버퍼·커서/선택이 복원, 미저장 변경 유실 없음
- **검증 AC**: AC4.1

### 시나리오 2: Claude Code 세션 복원 (백엔드 강제 종료)

- **사전 조건**: 진행 중인 Claude Code 세션(대화/컨텍스트)
- **실행 단계**: backend 강제 종료
- **기대 결과**: 세션이 마지막 상태로 복원되어 작업 이어가기 가능
- **검증 AC**: AC4.2

### 시나리오 3: 터미널 화면 복원 (백엔드 강제 종료)

- **사전 조건**: 여러 명령을 실행해 스크롤백이 쌓인 터미널
- **실행 단계**: backend 강제 종료 후 화면 확인 → 재연결 시도
- **기대 결과**: 마지막 화면·스크롤백(명령/결과)이 복원되어 표시되고, 새 PTY로 재연결 가능
- **검증 AC**: AC4.3

### 시나리오 4: 브라우저 탭 복원 (앱 강제 종료)

- **사전 조건**: 여러 탭(각기 다른 URL, 특정 순서)이 열린 브라우저
- **실행 단계**: Tessera 앱 프로세스를 강제 종료(kill) 후 재시작
- **기대 결과**: 탭별 URL과 탭 순서가 복원됨
- **검증 AC**: AC4.4

### 시나리오 5: 컨테이너 소멸 후에도 상태 보존

- **사전 조건**: container workspace에 편집기/Claude Code/터미널 상태 존재
- **실행 단계**: 컨테이너를 제거(rm)한 뒤 동일 workspace 복원 시도
- **기대 결과**: 호스트 측 영속 저장소에 상태가 남아 복원 가능(컨테이너 소멸과 독립)
- **검증 AC**: AC4.5

### 시나리오 6: 재연결 시 충돌 처리

- **사전 조건**: 복원 직후, 디스크 파일/세션이 그사이 외부에서 변경된 상황
- **실행 단계**: backend에 재연결
- **기대 결과**: 불일치가 감지·고지되고, 최신 상태 동기화 또는 사용자 선택으로 손실 없이 정리
- **검증 AC**: AC4.6
