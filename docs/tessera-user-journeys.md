# Tessera 사용자 여정 — 인덱스

> 이 문서는 사용자 여정의 **개요/인덱스**다. 각 여정의 전체 단계 흐름은 여정별 파일에 담겨 있다.
> 모든 여정은 `tessera-values.md`의 가치(V1~V4) 중 하나 이상을 달성하며, 각 단계는 어떤 mockup으로
> 시각화되는지를 추적한다. 최상위 판단 기준은 가치 문서다.

> **문서 체계 상 위치**: 가치 → PRD → AC → 테스트의 백엔드 사슬은 이미 갖춰져 있고,
> 사용자 여정 → mockup ↔ 디자인 시스템의 프론트엔드 사슬에서 여정이 **첫 문서군**이다.
> 현재 디자인 시스템과 mockup은 아직 없으므로, 모든 여정의 시각화 칸은 **미작성(⬜)** 상태다.

---

## 주 페르소나

이 문서의 여정들은 하나의 주 페르소나가 서로 다른 상황에 놓인 모습이다.

- **누가**: macOS에서 일하는 개발자.
- **평소 상황**: 터미널·인터넷 브라우저·텍스트 편집기·Claude Code(AI 코딩) 사이를 하루에도 수십 번 오가며
  작업한다. 앱 간 전환 비용, 환경 설정의 번거로움, 그리고 예기치 못한 종료로 인한 작업 손실에 민감하다.
- **무엇을 원하나**: 흩어진 도구를 하나의 표면에서 다루고(V1), 작업 성격에 맞는 실행 환경을
  자유롭게 고르며(V2), 격리 환경에서도 인증이 끊기지 않고(V3), 어떤 비정상 종료에도 작업을 잃지 않기를(V4) 원한다.

각 여정 파일은 이 개발자를 해당 상황에 놓고 목표 달성 경로를 기술한다.

## 시각화 상태와 다음 단계

현재 모든 여정(J1~J4)의 단계는 mockup이 없다(⬜). 프론트엔드 사슬이 막 시작되었기 때문이며,
구조적으로 다음 공백이 존재한다.

- **시각화 누락 단계(unvisualized step)**: J1~J4의 전 단계(24/24)가 어떤 mockup도 가리키지 못함.
- **디자인 시스템 부재**: UI 시각 언어(토큰/컴포넌트/패턴)가 아직 정의되지 않음.
- **mockup 인덱스 부재**: mockup ↔ 가치/여정/디자인 시스템 연결의 단일 소스가 없음.
- **시각화 없는 가치(unvisualized value)**: V1~V4 모두 아직 어떤 mockup으로도 시각화되지 않음.

권장 진행 순서(`design-doc-structure-validator`의 협업 가이드 기준):

1. **디자인 시스템 셋업** — 토큰/컴포넌트/패턴 정의(위치는 레포가 결정, 예: `docs/design-system/`).
2. **mockup 작성** — 각 여정 파일의 `M-Jx-Sn` 제안 ID에 대응하는 화면을 `web-artifacts-builder`로 제작.
3. **mockup 인덱스 작성** — 각 mockup이 어느 여정 단계/가치를 시각화하고 어떤 디자인 시스템 항목을 쓰는지 기록.
4. **여정 파일 갱신** — 각 단계의 시각화 칸을 ⬜ → ✅ 로 연결하고, 상태 추적 문서를 재검증.
