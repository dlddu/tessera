# PRD: 상태 복원 (State Restoration / Crash Resilience)

- **PRD ID**: PRD-4
- **참조 가치 문서**: `tessera-values.md`

## 달성 가치

- **V4: 작업 손실 없는 복원력** — 백엔드/앱 비정상 종료 후에도 편집기·Claude Code·터미널·브라우저의
  마지막 상태를 복원하여 작업 손실을 방지한다.

## 개요

**두 가지 종료 시나리오를 구분한다.**

- **백엔드 비정상 종료(#8, #9)**: 컨테이너가 죽거나 호스트 백엔드 프로세스가 비정상 종료된 경우.
  이때 Electron 앱 자체는 살아 있을 수 있으므로, 앱이 보유한 마지막 상태로 화면을 유지·복원하고 재연결을 제공한다.
- **애플리케이션 비정상 종료(#10)**: Tessera 앱 전체가 죽은 경우. 앱 메모리는 사라지므로, 상태가
  **디스크에 영속**되어 있어야 재시작 후 복원할 수 있다.

**영속 저장 위치 원칙**: 모든 복원용 상태는 **호스트 측 영속 저장소**에 주기적으로(디바운스) 기록한다.
컨테이너가 소멸해도 상태가 함께 사라지지 않도록, 상태 저장은 backend 수명과 독립이어야 한다.

| 대상 | 저장 내용 | 복원 트리거 |
|---|---|---|
| 텍스트 편집기 | 열린 파일, 미저장 버퍼, 커서/선택 | 백엔드 종료(#8) |
| Claude Code GUI | 마지막 세션 상태(대화/컨텍스트) | 백엔드 종료(#8) |
| 터미널 | 마지막 화면 + 스크롤백(명령/결과) | 백엔드 종료(#9) |
| 브라우저 | 탭별 URL(및 탭 순서) | 앱 종료(#10) |

## Acceptance Criteria

### AC4.1: 백엔드 종료 시 편집기 상태 복원

- **설명**: 백엔드가 비정상 종료되면, 편집기의 열린 파일 목록·미저장 버퍼 내용·커서/선택 위치를 복원한다.
  미저장 변경분이 종료로 인해 유실되지 않는다.
- **달성 가치**: V4
- **검증 방법**: 미저장 변경이 있는 편집기 상태에서 backend를 강제 종료한 뒤, 파일·버퍼·커서가 복원되는지 확인.

### AC4.2: 백엔드 종료 시 Claude Code 세션 상태 복원

- **설명**: 백엔드가 비정상 종료되면, Claude Code GUI의 마지막 세션 상태(대화/작업 컨텍스트)를 복원한다.
  사용자는 종료 직전 지점에서 작업을 이어갈 수 있다.
- **달성 가치**: V4
- **검증 방법**: 진행 중인 Claude Code 세션 상태에서 backend를 강제 종료한 뒤, 세션이 마지막 상태로 복원되는지 확인.

### AC4.3: 백엔드 종료 시 터미널 마지막 화면 복원

- **설명**: 백엔드가 비정상 종료되면, 터미널의 마지막 화면과 스크롤백(실행한 명령과 그 결과)을 복원해 표시한다.
  PTY는 사라졌으므로 복원된 화면은 읽기 가능한 상태로 보존되며, 사용자에게 재연결(새 PTY) 수단을 제공한다.
- **달성 가치**: V4
- **검증 방법**: 여러 명령을 실행해 스크롤백이 쌓인 터미널에서 backend를 강제 종료한 뒤, 마지막 화면·스크롤백이 복원되고 재연결이 가능한지 확인.

### AC4.4: 앱 종료 후 브라우저 탭별 URL 복원

- **설명**: Tessera 앱이 비정상 종료된 뒤 재시작하면, 브라우저의 각 탭 URL(및 탭 순서)을 복원한다.
- **달성 가치**: V4
- **검증 방법**: 여러 탭을 연 상태에서 앱을 강제 종료(프로세스 kill)한 뒤 재시작 시, 탭별 URL이 복원되는지 확인.

### AC4.5: 호스트 측 영속 저장 (backend/app 수명과 독립)

- **설명**: 위 복원 대상 상태는 호스트 측 영속 저장소에 주기적으로 저장되어, 컨테이너 소멸이나 앱 종료와
  무관하게 보존된다. 저장은 성능 영향을 줄이도록 디바운스/증분 기록한다.
- **달성 가치**: V4
- **검증 방법**: 컨테이너를 제거(rm)한 뒤에도 해당 workspace의 편집기/Claude Code/터미널 상태가 호스트 저장소에 남아 복원 가능한지 확인.

### AC4.6: 복원 상태와 재연결 간 일관성 처리

- **설명**: 복원된 상태로 backend에 재연결할 때, 충돌·불일치(예: 디스크 파일이 그사이 변경됨)를 감지하고
  사용자가 인지·선택할 수 있게 처리한다(최신 상태로 동기화 또는 사용자 선택).
- **달성 가치**: V4
- **검증 방법**: 복원 후 재연결 시점에 파일/세션 불일치를 유발하고, 충돌이 감지·고지되며 손실 없이 정리되는지 확인.

## 의존 관계

- PRD-1(AC1.5): 레이아웃 골격 직렬화 위에 컴포넌트 콘텐츠 상태 복원을 얹는다.
- PRD-2: 호스트 측 영속 저장이 가능하려면 backend 추상화가 상태 캡처 훅을 제공해야 한다.
- **경계 메모**: #8/#9는 "백엔드" 종료, #10은 "애플리케이션" 종료를 다룬다. 두 경로의 저장/복원 트리거가 다름에 유의.
