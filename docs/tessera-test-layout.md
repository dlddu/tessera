# 테스트 문서: 컴포넌트 & 레이아웃 시스템

- **Test ID**: T-1
- **대상 PRD**: PRD-1 (`tessera-prd-layout.md`)

## 검증 대상 AC

- AC1.1: 4종 컴포넌트를 pane/tab에 배치
- AC1.2: window 분할로 pane 생성 및 비율 조절
- AC1.3: tab 전환·재정렬·pane 간 이동
- AC1.4: 키보드 중심 레이아웃 조작
- AC1.5: 레이아웃 직렬화 및 재구성
- AC1.6: 포커스된 pane 일시 전체화면 토글
- AC1.7: 워크스페이스 목록 표시 및 전환

## 테스트 시나리오

### 시나리오 1: 4종 컴포넌트 배치

- **사전 조건**: 빈 window가 열린 workspace 1개
- **실행 단계**: 터미널 → 브라우저 → 텍스트 편집기 → Claude Code GUI 순으로 tab을 각각 생성
- **기대 결과**: 4종 surface가 모두 렌더되고 각자 기본 동작(입력/네비게이션/편집/세션)이 가능
- **검증 AC**: AC1.1

### 시나리오 2: 중첩 분할과 비율 조절

- **사전 조건**: 단일 pane window
- **실행 단계**: 수직 분할 → 각 pane을 수평 분할(2×2) → pane 경계 드래그/단축키로 비율 조절
- **기대 결과**: 2×2 레이아웃이 유지되고 비율 조절 후에도 깨짐·겹침 없음
- **검증 AC**: AC1.2

### 시나리오 3: tab 이동과 pane 정리

- **사전 조건**: 2개 pane, 각 pane에 tab 2개
- **실행 단계**: 한 pane의 tab을 다른 pane으로 드래그 이동 → 순서 변경 → 한 pane의 모든 tab 종료
- **기대 결과**: tab이 정확히 이동·재정렬되고, 비워진 pane은 정리되며 레이아웃이 재배치됨
- **검증 AC**: AC1.3

### 시나리오 4: 키보드만으로 레이아웃 구성

- **사전 조건**: 단일 pane window, 마우스 미사용
- **실행 단계**: 단축키로 2×2 분할 → pane 포커스 이동 → 각 pane에서 tab 생성·전환
- **기대 결과**: 마우스 없이 전체 시나리오 완수, 단축키 동작이 일관됨
- **검증 AC**: AC1.4

### 시나리오 5: 레이아웃 저장 후 재구성

- **사전 조건**: 복잡한 레이아웃(다중 pane·다중 tab·혼합 컴포넌트)
- **실행 단계**: 레이아웃 저장 → 앱 정상 종료 → 재시작
- **기대 결과**: 동일한 pane/tab 골격과 각 tab의 컴포넌트 타입이 그대로 재구성됨(콘텐츠 복원은 T-4 범위)
- **검증 AC**: AC1.5

### 시나리오 6: pane 일시 전체화면 토글과 가역 복귀

- **사전 조건**: 2×2 레이아웃(4개 pane, 임의 비율), 한 pane이 포커스됨
- **실행 단계**: 포커스된 pane을 단축키로 확대(zoom) → 확대 상태에서 tab 전환 → 단축키(또는 Esc)로 복귀
- **기대 결과**: 확대 시 해당 pane이 window 전체를 채우고 나머지 pane은 숨겨지며 zoom 배지가 표시됨. 복귀 시
  이전 2×2의 분할 구조·pane 비율·각 pane의 tab 구성이 변경 없이 그대로 복원됨(확대는 레이아웃 골격을 바꾸지 않음)
- **검증 AC**: AC1.6

### 시나리오 7: 워크스페이스 목록에서 전환과 상태 보존

- **사전 조건**: 서로 다른 레이아웃을 가진 workspace 2개 이상(예: A는 2×2, B는 단일 pane). 단일 앱 창에 workspace 목록이 표시됨
- **실행 단계**: 목록에서 workspace B를 마우스로 선택 → 단축키(예: ⌘+숫자)로 workspace A로 전환 → A에서 pane/tab을 조작한 뒤 B로 갔다가 다시 A로 복귀
- **기대 결과**: 선택 즉시 활성 window가 해당 workspace의 레이아웃으로 전환되고, 비활성 workspace의 pane/tab 구성·상태가
  보존되어 되돌아오면 그대로 복원됨. 마우스·단축키 양쪽으로 전환이 동작하며, 두 workspace의 window가 동시에 나란히
  표시되지는 않음(전환은 표시 대상만 바꿈)
- **검증 AC**: AC1.7
